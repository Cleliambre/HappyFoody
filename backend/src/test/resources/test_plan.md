# Plan de test du backend - HappyFoody

## Tests CRUD

| Fonctionnalité                                     | Objectif du test                                                          | Étapes / Entrées                                                     | Résultat attendu                                                                      | Automatisé ? |
|----------------------------------------------------|---------------------------------------------------------------------------|----------------------------------------------------------------------|---------------------------------------------------------------------------------------|--------------|
| **Création d’un compte utilisateur**               | Vérifier l’ajout correct d’un nouvel utilisateur à la base de données     | `POST /api/compte` avec un JSON valide                               | **200 OK** + corps contenant les infos du nouvel utilisateur                          | En cours     |
| **Modification d'un compte utilisateur**           | Vérifier la mise à jour correcte d'un utilisateur existant                | `PUT /api/compte/{id}` avec un JSON contenant les nouvelles données  | **200 OK** + corps contenant les infos mises à jour                                   | En cours     |
| **Suppression d'un compte utilisateur**            | Vérifier la suppression correcte d'un utilisateur de la base              | `DELETE /api/compte/{id}`                                            | **200 OK** + utilisateur supprimé de la base                                          | En cours     |
| **Accès aux informations d'un compte utilisateur** | Vérifier la récupération correcte des informations d’un utilisateur       | `GET /api/compte/{id}`                                               | **200 OK** + JSON contenant les infos de l’utilisateur                                | En cours     |
| **Récupération de tous les comptes utilisateurs**  | Vérifier que la liste complète des utilisateurs est renvoyée correctement | `GET /api/compte/all`                                                | **200 OK** + JSON contenant une liste d’utilisateurs (même vide si aucun utilisateur) | En cours     |
|                                                    |                                                                           |                                                                      |                                                                                       |              |
| **Création d’une recette**                         | Vérifier l’ajout correct d’une nouvelle recette à la base de données      | `POST /api/recette` avec un JSON valide                              | **200 OK** + corps contenant les infos de la nouvelle recette                         | En cours     |
| **Modification d'une recette**                     | Vérifier la mise à jour correcte d'une recette existante                  | `PUT /api/recette/{id}` avec un JSON contenant les nouvelles données | **200 OK** + corps contenant les infos mises à jour                                   | En cours     |
| **Suppression d'une recette**                      | Vérifier la suppression correcte d'une recette de la base                 | `DELETE /api/recette/{id}`                                           | **200 OK** + recette supprimé de la base                                              | En cours     |
| **Accès aux informations d'une recette**           | Vérifier la récupération correcte des informations d’une recette          | `GET /api/recette/{id}`                                              | **200 OK** + JSON contenant les infos de la recette                                   | En cours     |
| **Récupération de toutes les recettes**            | Vérifier que la liste complète des recettes est renvoyée correctement     | `GET /api/recette/all`                                               | **200 OK** + JSON contenant une liste de recettes (même vide si aucune recette)       | En cours     |
